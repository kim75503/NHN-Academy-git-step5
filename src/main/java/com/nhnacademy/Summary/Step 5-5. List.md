# Step 5-5. List

## 리스트(List) 자료구조
### 리스트란?

리스트는 순서가 있는 데이터의 나열을 저장하는 자료구조입니다. 배열도 리스트의 한 종류입니다.

리스트를 구현하는 방식에 따라 장단점이 다릅니다:

리스트 (순차 자료구조)
├── 배열 (Array) - 연속된 메모리 공간
└── 연결 리스트 (Linked List) - 노드들의 연결

## 1단계: 배열 - 연속 메모리 기반 리스트

### 배열은 데이터를 연속된 메모리 공간에 저장합니다.

메모리 주소:  100     104     108     112
           ┌───────┬───────┬───────┬───────┐
           │ "Java"│ "운동" │ "독서" │       │
           └───────┴───────┴───────┴───────┘
              [0]     [1]     [2]     [3]
### 장점: 인덱스로 즉시 접근 가능 → O(1)
 - arr[2]는 시작 주소 + (2 × 크기)로 바로 계산

### 단점: 크기가 고정됨

 - 요소 추가/삭제 시 새 배열 생성 & 복사 필요

String[] arr = new String[3];  // 크기 3으로 고정!
arr[3] = "영화";  // ❌ ArrayIndexOutOfBoundsException

## 2단계: 연결 리스트 - 노드 연결 기반 리스트
 - 연결 리스트는 각 요소(노드)가 다음 노드의 위치를 기억하는 방식입니다.

┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ data: "Java" │    │ data: "운동"  │    │ data: "독서"  │
│ next: ───────┼───▶│ next: ───────┼───▶│ next: null   │
└──────────────┘    └──────────────┘    └──────────────┘
     [0]                 [1]                 [2]
### 장점: 크기가 가변적

- 새 노드를 연결만 하면 됨 (복사 불필요)

새 요소 "영화" 추가:
      ┌──────────────┐                         ┌──────────────┐
... ─▶│ data: "독서"  │                         │ data: "영화"  │
      │ next: ───────┼────────────────────────▶│ next: null   │
      └──────────────┘                         └──────────────┘
           [2]                                      [3] ✅
### 단점: 인덱스로 즉시 접근 불가 → O(n)
 - [2]번 요소를 찾으려면 [0] → [1] → [2] 순서로 따라가야 함

## 3단계: Java의 선택 - ArrayList vs LinkedList
 - Java는 두 가지 장단점을 모두 활용할 수 있도록 두 가지 구현을 제공합니다:

특성	         ArrayList	        LinkedList
내부 구조	      배열	                 노드 연결
인덱스 접근	      빠름 O(1)	           느림 O(n)
끝에 추가	     빠름 (대부분)	       빠름
중간 삽입/삭제	느림 (이동 필요)          빠름 (연결만 변경)
메모리	      효율적	           노드당 추가 공간 필요

### 선택 기준:
 - 대부분의 경우 → ArrayList (인덱스 접근이 빈번)
 - 중간 삽입/삭제가 매우 빈번 → LinkedList

+ 일반적인 상황에서는 ArrayList를 사용하세요. 성능 차이가 문제될 때만 LinkedList를 고려합니다.
